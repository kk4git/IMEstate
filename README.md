# IMEstate
码农的五笔
## Windows 11 自带五笔输入法
作为一名码农老登，五笔输入法是我每天敲代码、写文档的命根子。Windows 11 自带的这玩意儿，整体还算中规中矩 —— 系统整合度高，稳定性也凑合，还支持五笔拼音混输。平时用着，基本不掉链子。

但！它的问题也够让人抓狂的:

### 吐槽时间: 没有固定作用的内裤

1. **默认全角标点，祖传级恶习**  
   源码里混进一个全角句号、逗号，那画面太美我不敢看——找半天bug，改半天才发现是标点在捣鬼。更气人的是：你好不容易用 Ctrl+. 切到半角了，结果一切换窗口、或者输入法一抖，它又默默变回全角！  
   拼音输入法好歹还能设置，它倒好，只要你切到中文状态，标点就铁了心要全角。码农的血压，直接原地起飞。

2. **中文输入法里的“英文模式”，脱裤子放屁**  
   我辛辛苦苦按 Win+Space 或者 Ctrl+Shift 切到中文输入法，结果一敲键盘还是英文……大哥，你这不是多此一举吗？我要输入英文，直接关输入法不就完了？非得让我在中文状态下打英文，这是哪门子设计？

3. **全角英文，存在即合理？**  
   不管你是"英"还是"中"，敲出来的字母永远是宽体大胖子。谁能告诉我这功能到底救了谁的命？ 
   好在它还算有记性，不会像全角标点那样当仁不让。

一句话总结：这输入法就像一条没有固定作用的内裤 —— 能遮羞，但松松垮垮的，里面的东西动不动就跑偏，一不小心还会扯着蛋。

### 我的诉求
很简单，中文输入法就该**明确区别于**英文输入：  
- 关了，就是纯正的英文键盘；  
- 开了，就该是标准的中文输入状态。

你非要说“中文输入就该全角标点”，那也行，但能不能至少**记住我的标点设置**？  
实在记不住，也**给我个选项**，让我一打开输入法就默认半角标点行不行？  
至于全角英文那玩意儿，真的没几个人天天用吧？默认关掉它，能死吗？

总之：**我想要一个可预期的输入法状态，最好让我自己定义这个“预期”！**

### 解决方案
市面上外挂五笔确实有很多更香的，但我这人懒，能用系统自带的就不想多装一个软件。关键是怎么避免每次切换时，手忙脚乱地狂按 Win+Space、Ctrl+Shift、Shift+Space、Ctrl+.、Ctrl+Space？

不换输入法，就用 **AutoHotkey** 神器整活  
简单粗暴的脚本就能实现：CapsLock 一键切换中英文，顺便把中文输入法自动切到半角标点。

```ahk
CapsLock:: {
  Send "#{Space}"
  Sleep(100)
  Send "{Ctrl Down}.{Ctrl Up}"
}
```
听起来很完美，对吧？但实际用起来，就是会翻车： 
有时切过来显示的是"英"，你得再手动 Ctrl+Shift 切到"中"； 
有时明明是半角标点，一切"中"结果秒变全角……老哥们血压又上来了。 

### 终极救星：
有没有一种工具，能**实时检测**当前输入法状态？  
- 检测到"英"就自动 Ctrl+Space 切中文；  
- 检测到全角标点就自动 Ctrl+. 切半角；  
- 检测到全角英文就 Shift+Space 切半角。

### 说干就干:
网上一搜，IMM有这功能！代码如下：
```C
#include <Windows.h>
#pragma comment (lib, "imm32.lib")
#pragma comment (lib, "user32.lib")
int main() {
    HWND hIME = ImmGetDefaultIMEWnd(GetForegroundWindow());
    LRESULT ime = SendMessage(hIME, WM_IME_CONTROL, 5, 0), s = SendMessage(hIME, WM_IME_CONTROL, 1, 0);
    bool bCn = s & 1, bWide = s & 8, bFull = s & 1024;
    return ime? (bCn+1) * 100 + (bWide+1) * 10 + bFull+1: -1;
}
```
编译运行，看着还真像那么回事。赶紧加入 AutoHotkey 脚本。  
结果...寄了，状态随便乱飞，根本没法用。  
又是一通网上搜索+AI问答，结果说 Windows 11 下 IMM 这东西就是不可靠的。

那怎么办呢？有没有稳定可靠的办法呢？

有的！兄弟，有的！

## IMEstate
我的开源小工具，就是干这个的：
- 通过窗口特征定位 IME 窗口；  
- 读取窗口内容，检测是否处于中文输入法状态；  
- 如果是，就识别状态栏上的3个小图标（中/英、全半角英文、全半角标点），并返回状态码。

### 返回值
超级直观，百位对应“英”/“中”，十位对应全角英文，个位对应全角标点：  
-  -1：IME 没打开或检测失败  
- 212：中文 + 半角英文 + 全角标点（默认地狱模式）
- 211：中文 + 半角英文 + 半角标点（我的完美状态）
- 111：英文 + 半角英文 + 半角标点（常见翻车现场）  
- 121：英文 + 全角英文 + 半角标点（外星翻车现场）  
- ...：其他可能的组合

### 脚本
整合 AutoHotkey 后，CapsLock 直接升级成“智能一键完美切换”：

```ahk
EXE := "IMEstate.exe" ;状态检测器

ToIME() { ;打开/关闭IME
  Send "#{Space}"
}
ToHLF() { ;切换半/全角标点
  Send "{Ctrl Down}.{Ctrl Up}"
}
ToCHN() { ;切换中/英输入状态
  Send("{Ctrl Down}{Space}{Ctrl Up}")
}
ToNRW() { ;切换窄/宽英文
  Send("{Shift Down}{Space}{Shift Up}")
}

WaitIME() { ;等待IME打开
  Loop 10 { ;最多100ms
    if (RunWait(EXE) > 0)
      return 1
    Sleep(10)
  }
  return -1
}
WaitCHN() { ;等待中文输入状态
  Loop 10 { ;最多100ms
    if (2 == Mod(RunWait(EXE) // 100, 10))
      return 1
    Sleep(10)
  }
  return -1
}

CapsLock:: { ;关闭/打开中文输入, 同时换掉默认的全角标点
  old := RunWait(EXE)
  ToIME()
  if (old > 0) ;关闭IME, 直接结束 
    Exit
  if (WaitIME() < 0) ;打开IME失败
    Exit

  new := RunWait(EXE)
  if (2 == Mod(new//10, 10)) ;全角英文
    ToNRW()

  if (1 == Mod(new//100, 10)) { ;英文IME模式
    ToCHN()
    if (WaitCHN() < 0) ;换到中文IME失败
      Exit
  }

  if (2 == Mod(RunWait(EXE)//1, 10)) ;全角标点
    ToHLF()
}
```

### 额外福利
想快速打中文标点？再加几行热键就行：
```ahk
; Alt + , . ; \  快速打全角标点
!,::，
!;::；
!\::、
!.::。
```

### Debug 模式，程序员专属

程序支持 `-v`、`-vv`、`-vvv` 参数，控制台会输出详细调试信息。最狠的 `-vvv` 甚至会告诉你每个图标是怎么识别出来的，方便你排查不同 DPI、不同分辨率下的问题。
